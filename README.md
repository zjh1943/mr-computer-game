# 电脑先生

这是一个和我家正在上二年级的小孩一起做的小项目。

整个项目大约花了 5 到 6 个小时，一边想一边改，一边试一边加。最重要的是，项目里的想法、功能、特性，基本都是他自己一点一点提出来的。他全程都在不断往里面加新功能、加新玩法，我们就这样把一个会说话的小电脑，慢慢做成了一个可以互动的小网页游戏。

## 项目简介

`电脑先生` 是一个运行在网页里的互动小游戏。玩家可以和电脑先生聊天、喂食、拖拽、切换白天夜晚、进入恐怖之夜，还能看到很多细小的互动反应。

## 主要特性

- 中文聊天互动，电脑先生会根据输入内容回复。
- 支持浏览器语音识别和语音播报。
- 平时会显示笑脸，聊天时会显示字幕。
- 字幕大多数是白色，偶尔会变成彩色。
- 有白天和夜晚场景，而且会自动切换。
- 太阳会摇动、会看向电脑先生。
- 月亮会上下切换，白云会飘动。
- 草地、树、桌子、键盘、主机、电线组成完整场景。
- 恐怖之夜模式可以通过聊天或按钮开启和关闭。
- 恐怖之夜里天空会变深红，云会变红，还会长出会看的眼睛。
- 恐怖之夜里太阳会变成一只大眼睛。
- 恐怖之夜里电脑先生会戴上面具，帽子会消失。
- 恐怖之夜状态不会自动耗电。
- 如果恐怖之夜结束，场景会恢复正常。
- 电脑先生有电量系统，会掉电、低电量、没电。
- 长时间不喂食会进入低电量，严重时会掉下去。
- 低电量聊天时会直接恢复到 100 电量。
- 没电时聊天也能把它重新唤醒。
- 没电时把食物放到帽子上，也能把它唤醒。
- 喂食后会说“好吃”，还会显示电量。
- 食物区里有苹果、香蕉、橘子、饼干。
- 食物可以拖动，不会因为放到地上就消失。
- 掉到地上的食物还能重新拿起来继续喂。
- 苹果和橘子放到草地上会长成小树。
- 小树上会结果子。
- 小树可以被强行拔掉，拔掉后会重新变成食物。
- 拔下来的食物还能继续喂给电脑先生。
- 鼠标移动时，电脑先生的眼睛会跟着看鼠标。
- 轻轻摸它的时候，帽子会自己飞起来一下，但主机不会飞。
- 可以直接拖动电脑先生，把它拽上天。
- 如果从高处掉下来，它会摔晕，歪在桌子上。
- 摔晕时眼睛会变成很多圈圈。
- 掉下来以后它会吐，严重时还会吐彩虹。
- 如果太久不和它玩，它会犯困，闭上眼睛，嘴巴也会变小。
- 帽子可以单独拿下来，放到地上。
- 左下角有恐怖之夜按钮，右下角有白天夜晚按钮。

## 运行方式

这是一个纯前端静态项目，直接用本地静态服务器打开即可，例如：

```bash
python3 -m http.server 4173
```

然后访问：

`http://127.0.0.1:4173/`

## 文件说明

- `index.html`：页面结构
- `styles.css`：视觉样式和动画
- `app.js`：交互逻辑、聊天逻辑、拖拽逻辑、场景切换逻辑

## 共创过程记录

这一部分专门记录，这个项目是怎么被一个二年级小朋友一点一点“想出来、说出来、加出来”的。

整个过程不是先写完整策划，再开始开发，而是非常典型的儿童式创作流程：

1. 先有一个很简单的核心想法。
   一开始只是想做一个“电脑先生”，放在屏幕里，可以和人聊天。

2. 很快开始加入角色外形。
   小朋友最先在意的不是代码，而是“长什么样”。所以很快就开始提出：
   - 要像节奏盒子里的电脑先生
   - 要戴帽子
   - 要有竹蜻蜓
   - 要有桌子、草地、树、云、太阳、月亮

3. 接着开始要求角色“活起来”。
   他不满足于一个静态角色，而是不断要求：
   - 会说话
   - 嘴巴会动
   - 眼睛会左右看
   - 字幕和笑脸轮流出现
   - 鼠标动的时候眼睛要跟着看

4. 然后开始出现“规则”和“状态”。
   这一步特别像真正的游戏设计。小朋友开始自然提出：
   - 白天和夜晚切换
   - 电量系统
   - 没电会掉下来
   - 喂食可以恢复
   - 不聊天的时候会困

5. 再往后就进入“特殊模式”的设计。
   例如：
   - 恐怖之夜
   - 云长眼睛
   - 太阳变大眼睛
   - 电脑先生戴面具
   - 深红色天空

6. 最后进入“玩具式交互”和“连锁反应”阶段。
   这是这个项目最有意思的地方。小朋友开始把它当成一个可以摆弄的玩具，而不是单纯聊天页面：
   - 可以拖动电脑先生
   - 可以拖动帽子
   - 可以把食物放到地上
   - 食物会长成树
   - 可以把树拔掉再喂回去
   - 摔下来会晕、会吐、还会吐彩虹

从学习角度看，这个过程很值得参考：  
不是“先有完整方案再做”，而是“先做一个能看见的东西，然后不断追问：还能不能更像、更活、更好玩”。

## 提示词演变记录

下面把这次提示词演变整理成表格。不是逐字逐句的完整对话日志，而是按阶段归纳后的学习版记录，重点展示这个游戏是如何被一个二年级小朋友一步一步“加出来”的。

| 阶段 | 当时最关心的问题 | 典型提示词方向 | 这一阶段做出了什么 |
| --- | --- | --- | --- |
| 第一阶段：先有主角 | “先有一个能聊天的电脑先生” | 做一个游戏；场景里有电脑先生；把电脑先生放在屏幕上；屏幕上的内容可以和他聊天；最底下是发送语音聊天的地方；要做成中文的 | 先确定了项目的核心：这是一个以“电脑先生”为主角、可以聊天的中文互动网页。 |
| 第二阶段：把角色长相定下来 | “它要长得像我脑子里的那个角色” | 把电脑先生缩小一点；屏幕最大；做得像很老式的电脑；戴帽子；头顶有竹蜻蜓；桌子要有腿；站在草地上；背景要有蓝天和白云 | 角色从抽象概念变成了具体形象，开始有老式电脑、帽子、桌子、草地、蓝天白云这些明确外观。 |
| 第三阶段：让角色活起来 | “它不能只是站着，它得会动” | 笑脸要左右看；嘴巴要一张一合；有时候显示字幕，有时候显示笑脸；太阳会看电脑先生；云会飘；月亮会升起来，太阳会落下去 | 游戏从静态画面进入动态阶段，角色、太阳、云和月亮都开始有了动作。 |
| 第四阶段：加规则和资源 | “它要能被照顾，也要有状态变化” | 加一个食物区；喂它吃东西；吃完会说好吃；屏幕上显示电池；长时间不喂会低电量；没电会掉下来；聊天的时候可以恢复电量 | 开始出现真正的游戏系统，包括食物、电量、低电量、恢复机制和掉落反馈。 |
| 第五阶段：把世界做完整 | “不只是角色，整个世界也要变” | 白天和夜晚切换；白云飘久了会到晚上；白天有太阳公公；晚上有月亮；树要更多，有大有小；地面颜色夜晚要更深 | 游戏从“一个角色”扩展成“一个会变化的场景世界”，开始有时间感和氛围感。 |
| 第六阶段：设计特殊模式 | “要有一个特别吓人的版本” | 开启恐怖之夜；云会长眼睛；太阳变成一只大眼睛；天空变深红色；电脑先生戴面具；只有取消恐怖之夜才会恢复 | 做出了完整的“恐怖之夜”模式，包含独立视觉规则、独立交互状态和手动关闭逻辑。 |
| 第七阶段：变成可以摆弄的玩具 | “我不只是想看，我还想拖它、拿它、种它” | 用鼠标摸摸它；帽子会飞起来一下；可以把电脑先生拖上天；掉下来以后会晕；帽子可以拿走；食物掉在地上不会消失；食物种下去会长成树；树能拔掉再喂回去；摔得太重会吐彩虹 | 游戏进入最自由的阶段，开始像一个可实验、可恶作剧、可反复操作的小玩具世界。 |

### 从这张表里能看出的创作规律

| 观察到的规律 | 表现方式 | 对别人有什么参考价值 |
| --- | --- | --- |
| 先做主角，再做世界 | 一开始只关心“电脑先生是谁”，后面才逐步加入草地、树、云、太阳、月亮 | 和孩子一起共创时，先把主角做出来，比一开始做完整世界更容易推进。 |
| 先追求像，再追求系统 | 前面大量提示词都在改大小、形状、帽子、桌子、屏幕比例 | 孩子通常先从视觉判断“像不像”，不是先想数据结构或程序架构。 |
| 动起来之后，灵感会变多 | 一旦嘴巴会动、眼睛会看、太阳会摇，后面的创意明显开始爆发 | 最早期尽快做出可见反馈，能更快激发下一轮需求。 |
| 游戏规则是被“玩”出来的 | 电量、喂食、睡觉、恐怖之夜、种树、拔树这些规则，都是在反复试玩时慢慢加出来的 | 不一定要先写完整设计文档，也可以先做原型，再从试玩里长出规则。 |
| 最有创造力的功能常常来自口语化表达 | 例如“云长眼睛”“帽子可以拿走”“摔下来会吐彩虹” | 不要低估孩子随口说出的话，这些往往正是最有想象力的交互点。 |

## 这次共创能给别人什么参考

如果别人也想参考这次做法，可以总结成几条非常实用的经验：

1. 不要一开始就逼孩子写完整策划。
   先让他看到一个能动的东西，再往上加。

2. 每次只改一个最想改的地方。
   例如先改脸，再改树，再改电量，再改恐怖之夜。

3. 允许反复推翻。
   小朋友经常会说“不是这样”“以前的不要”“要更像图片里那个”，这不是折腾，而是在逐步形成审美判断。

4. 把孩子的口语直接当需求。
   很多真正有创造力的功能，往往都来自看上去“不专业”的一句话，比如：
   - “摔下来会晕”
   - “帽子可以拿走”
   - “云彩会长眼睛”
   - “食物会长成树”

5. 先让作品好玩，再让结构完整。
   这次项目最成功的地方，不是先把工程搭得多规范，而是始终在追问：
   “它现在好不好玩？”

## 适合别人模仿的共创方式

如果你也想和孩子一起做类似项目，可以照这个顺序来：

1. 先定一个主角。
2. 再定主角长什么样。
3. 然后让它动起来。
4. 再给它加资源系统，例如电量、食物、状态。
5. 接着加场景变化，例如白天夜晚。
6. 最后再加特别模式和恶作剧式互动。

这种方式特别适合：

- 和孩子一起做第一个 AI 共创项目
- 让孩子体验“我说一句，世界就会变化”
- 用游戏方式练习表达、观察和创意组织
